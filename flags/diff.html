Edge cases & gotchas

found in the launch code + Chrome behavior
  • 01Chrome is last-wins for repeated switches. Values are replaced, never merged. LCNC's live agentic path merges all --disable-features into one Set-deduped switch for exactly this reason; a user flag repeating a base switch silently replaces it.
  • 02A second --enable-features clobbers Dark mode. BrowserFlagsHandler.addOrOverrideArg replaces valued switches by key, so any other flag emitting --enable-features=X overwrites WebContentsForceDark.
  • 03Camera + Auto-select share args. Both expand to --use-fake-ui/-device-for-media-stream; the handler dedupes bare switches so they appear once — the diff above mirrors that.
  • 04Fake-ui beats Block-all for camera. --use-fake-ui-for-media-stream auto-accepts before the permission prompt exists, so --deny-permission-prompts has nothing to deny — getUserMedia resolves fake even with Block-all on.
  • 05Dedupe never matches the baseline. User flag args are stripped of -- while baseline entries keep it, so a conflicting user flag is appended as a duplicate — and last-wins means the user flag wins.
  • 06en-US is invisible. The default language preset emits --lang=en-US… which is indistinguishable from not setting the flag at all. Test language with a non-default locale.
  • 07Chrome normalizes locales. --lang=hi-IN → navigator.language "hi", es-ES → "es" (not shipped UI locales); en-US/en-GB/zh-CN/ja/ko/ru/ar/fr/es survive verbatim.
  • 08Incognito is JS-invisible on Chrome 133+. storage.estimate() now returns usage + 10 GiB in both modes, deliberately. Verify incognito via the run's UI/video. It also flips LCNC's new-tab strategy (useJSScriptForOpeningNewTab).
  • 09Clipboard "Enabled" ≙ default. The agentic and desktop launches already set clipboard = 1 in default prefs — only Disabled (pref = 2) is observable.
  • 10The WDIO replay path ignores user flags. It force-sets geolocation/camera/mic/notifications = deny and appends --deny-permission-prompts on every run — Location/Clipboard flags simply don't reach it.
  • 11navigator.webdriver can lie. The group-level disableAutomationControlled suppression flag passes --disable-blink-features=AutomationControlled, making webdriver read false on an automated run.
  • 12Shipped typo + legacy switches. --disable-prompt-on-repos (sic) ships next to the correct -repost; --reduce-security-for-testing, --enable-fixed-layout and friends were removed from Chrome years ago — all silently ignored.
  • 13Mobile flags aren't consumed. The browserFlags.mobile bucket is not read anywhere on the agentic path (desktop-only today); "mobile" runs are desktop-Chrome mobileEmulation.
  • 14Un-prefixed keys can vanish. browser_flags.json stores keys without --; the BS /playwright?caps= endpoint silently drops un-prefixed args on the legacy pwCaps path (the live path re-normalizes).
